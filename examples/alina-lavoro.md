# examples/alina-lavoro.md — Alina Lavoro (Historical Example)

## What it is

Alina Lavoro is the historical advanced case that produced many of the lessons encoded in dev-method.
It is not the container of dev-method, and dev-method does not depend on it.

## Why it matters

Alina Lavoro was the first project to demonstrate, in practice:
- GitHub as source of truth and persistent AI memory
- LLM Wiki / docs routing for agents
- n8n as control plane
- Telegram as human gate
- Branch safety rules under autonomous iteration
- Autonomy hardening from Level 1 toward Level 3

These lessons were extracted, generalized, and encoded here.

## What Alina Lavoro is not

- It is not a reference implementation that importing projects must follow.
- It is not the canonical example of dev-method usage (GIS Tool is the first real pilot).
- Its project-specific patterns (GAS, spreadsheet structure, HR domain) do not generalize.

## How to read it

If you are encountering dev-method for the first time, you do not need to read Alina Lavoro.
It is useful context for understanding *why* certain rules exist, not what the rules are.
The rules are in `core/`.
